01
Abrir sem condescendência: o objetivo é genuinamente calibrar. A turma é mista — informáticos, mas com maturidades de dados muito diferentes. Estas cinco perguntas dizem-me se posso correr nos fundamentos (Introdução à Engenharia de Dados) ou se vale a pena demorar. Tomar nota mental das mãos por pergunta.
Pergunta 1 · a vossa stack

Com que sistemas de dados trabalham?

Bases relacionais

Postgres, MySQL, SQL Server, Oracle.

NoSQL

Mongo, Redis, Cassandra, Elastic.

Warehouses / lakes

BigQuery, Snowflake, Redshift, S3.

Big data / streaming

Spark, Hadoop, Kafka, Flink.

Mão no ar por cada um que usam no dia-a-dia.

02
Mapear a stack real. Quero perceber a escala: quem só toca em bases relacionais e quem já lida com volumes que exigem Spark/streaming. Se quase ninguém usa big data, não vale a pena puxar muito por esses exemplos; se a sala é sénior, posso ir mais fundo. Perguntar também qual o papel deles — dev de aplicação que toca na BD, analista, alguém de plataforma de dados.
Pergunta 2 · de onde vêm os dados

Mais bases de dados, ou mais ficheiros?

Bases de dados

Modelar e consultar

Esquemas, queries, transações, índices. O dado vive num sistema vivo.

SQL, modelação, performance
Ingestão de ficheiros

Receber e processar

CSV, Excel, Parquet, JSON, exportações e APIs. O dado chega de fora e é preciso limpá-lo.

ETL, parsing, validação

Onde passam mais tempo? (vale dividir a sala)

03
Esta distinção orienta muito a aula. Quem vive em bases de dados vai ligar-se mais ao bloco "construir com a GenAI" (SQL que mente, schema linking). Quem vive de ficheiros vai ligar-se ao "construir para a GenAI" (o LLM-como-ETL, extração de texto sujo). A maioria faz as duas coisas — mas o equilíbrio diz-me que exemplos pesam mais.

Qual descreve melhor o vosso dia-a-dia?

04
Termómetro de maturidade de engenharia de dados. Quem usa orquestrador já tem o vocabulário de DAG, idempotência, retries — posso ser breve nos fundamentos. Quem corre tudo à mão vai beneficiar do bloco de fundamentos por inteiro. É também a deixa para introduzir, mais tarde, porque a reprodutibilidade importa quando se mete um LLM (não-determinístico) no meio.
Pergunta 4 · ferramentas de IA

Já usaram ferramentas que geram código?

Autocompletar

Copilot, Tabnine — sugestões enquanto escrevem.

Chat para código

ChatGPT, Claude, Gemini no browser.

Editores agênticos

Cursor, Claude Code, Windsurf — agem no repositório.

App-builders

Lovable, v0, Bolt — da ideia à app.

Quais já experimentaram — e para quê?

05
Saber o ponto de partida com IA generativa. Há uma diferença grande entre quem já tem o Copilot ligado todos os dias e quem só experimentou o ChatGPT uma vez. Quem usa editores agénticos (Cursor, Claude Code) já viveu o "escrever → orientar → validar" — posso apoiar-me nisso. Perguntar para que usam: gerar boilerplate, traduzir entre linguagens, explicar código alheio, escrever testes. Liga ao bloco "construir com a GenAI".

06
Esta é a pergunta aberta, a mais valiosa. Deixar a sala falar. Recolher expetativas e ceticismos — vão servir de fio ao longo da aula e fechamos voltando a elas. Pontos a fazer emergir, sem os impor: o LLM brilha em texto livre e desambiguação, falha em garantias (determinismo, números, auditoria); o trabalho de dados por baixo é o que faz a diferença. Mas hoje só ouço — não corrijo.
