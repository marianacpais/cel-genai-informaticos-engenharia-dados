A GenAI ao serviço do engenheiro · rever o modelo gerado

O modelo "compila" — e está errado

gerado → receita_diaria.sql, produzido pelo copiloto
⟶ o que o LLM gera 
SELECT
  e.data::date AS dia,
  SUM(l.valor) AS receita
FROM encomendas e
JOIN linhas l
  ON l.enc_id = e.id
GROUP BY 1
-- (sem testes)

compila ✓ · receita inflada

⟶ a revisão de engenharia 
-- agregar linhas ANTES do join
-- (senão o fan-out duplica a receita)
-- + testes:
--   dia    → único, não-nulo
--   receita → não-nulo
-- + materialização incremental

testado ✓ · grão correto

O LLM gera o esqueleto, não o contrato. Faltam os testes, e o join repete a armadilha do fan-out. "Compila" não é "está certo".

33
[PENDENTE DE DECISÃO.] A armadilha do fan-out reaparece dentro de um modelo gerado — o copiloto não aprende a lição entre slides. A revisão acrescenta o que faz um modelo de produção: o teste de unicidade no grão, o não-nulo, e a materialização incremental. É exatamente o code review que vale mais, não menos.
